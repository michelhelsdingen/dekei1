## Critical: Database Strategy

The `dekei1` database EXISTS as a separate PostgreSQL database on the server, but PostgREST only serves the `postgres` database. Therefore:

**Chosen approach**: Create a `dekei1` schema WITHIN the `postgres` database. PostgREST supports multiple schemas — tables live at `public` by default but can be exposed in a custom `dekei1` schema by passing schema headers. The Supabase JS client supports this via `db.schema` option.

When creating tables via `pg/query` endpoint:
- First create the schema: `CREATE SCHEMA IF NOT EXISTS dekei1;`
- Create tables inside that schema: `CREATE TABLE dekei1.matches (...)`
- The `pg/query` endpoint runs against `postgres` database — so `dekei1.matches` refers to the schema named `dekei1` in the `postgres` database

```sql
-- Schema aanmaken in postgres database
CREATE SCHEMA IF NOT EXISTS dekei1;

-- Matches tabel
CREATE TABLE IF NOT EXISTS dekei1.matches (
  id SERIAL PRIMARY KEY,
  round INTEGER NOT NULL,
  match_date DATE NOT NULL,
  opponent TEXT NOT NULL,
  home_away TEXT NOT NULL CHECK (home_away IN ('home', 'away')),
  match_time TIME NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Availability tabel
CREATE TABLE IF NOT EXISTS dekei1.availability (
  id SERIAL PRIMARY KEY,
  match_id INTEGER NOT NULL REFERENCES dekei1.matches(id) ON DELETE CASCADE,
  player_name TEXT NOT NULL CHECK (player_name IN ('Mark', 'Coen', 'Bas', 'Maarten', 'Niels', 'Michel')),
  status TEXT NOT NULL DEFAULT 'unknown' CHECK (status IN ('yes', 'no', 'reserve', 'unknown')),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(match_id, player_name)
);
```

```sql
GRANT ALL ON ALL SEQUENCES IN SCHEMA dekei1 TO dekei1;
ALTER DEFAULT PRIVILEGES IN SCHEMA dekei1 GRANT ALL ON TABLES TO anon, authenticated, service_role;
ALTER DEFAULT PRIVILEGES IN SCHEMA dekei1 GRANT ALL ON SEQUENCES TO anon, authenticated, service_role;
```

**Stap 4: Supabase client aanmaken**

Maak `src/lib/supabase.ts`:

```typescript
import { createClient } from '@supabase/supabase-js'

const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL!
const supabaseAnonKey = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!

// Client geconfigureerd voor dekei1 schema in postgres database
// PostgREST bedient postgres database; dekei1 is een schema daarin
export const supabase = createClient(supabaseUrl, supabaseAnonKey, {
  db: {
    schema: 'dekei1',
  },
})
```

```typescript
import { supabase } from '@/lib/supabase'

export default async function Home() {
  const { data: matches, error } = await supabase
    .from('matches')
    .select('*')
    .order('round', { ascending: true })

  if (error) {
    return (
      <div className="p-8">
        <h1 className="text-2xl font-bold text-red-600">Database Error</h1>
        <pre className="mt-4 text-sm">{JSON.stringify(error, null, 2)}</pre>
      </div>
    )
  }

  return (
    <div className="p-8">
      <h1 className="text-2xl font-bold mb-4">DE KEI 1 — Foundation Check</h1>
      <p className="text-green-600 font-semibold mb-4">✓ Database verbinding werkt!</p>
      <ul className="space-y-2">
        {matches?.map((match) => (
          <li key={match.id} className="border p-3 rounded">
            <strong>Ronde {match.round}</strong>: {match.opponent} ({match.home_away}) — {match.match_date} {match.match_time}
          </li>
        ))}
      </ul>
    </div>
  )
}
```
